# Inference Provider Guide

AE v2 supports provider-agnostic generation so you can use Codex, OpenAI API, local models, or any other backend.

## Why This Is Important

- Avoid hard lock-in to one generation runtime.
- Keep AE generation behavior consistent across environments.
- Let teams swap providers without rewriting core AE logic.

## Audience

### For Humans

Use this guide when building a custom backend integration (service, SDK, gateway, local inference runtime).

### For Agents

Use this contract when implementing alternate generation providers in code updates.

## Core Contract

Use these types from `agentic_executables_core`:

- `InferenceClient` (`lib/src/ports/inference_client.dart`)
- `InferenceRequest` / `InferenceResponse` (`lib/src/models/inference.dart`)
- `InferenceGenerationEngine` (`lib/src/adapters/inference_generation_engine.dart`)

`InferenceGenerationEngine` transforms structured model output into AE files and enforces the required file set.

## Required Output Shape

Provider output must include:
- `ae_install.md`
- `ae_uninstall.md`
- `ae_update.md`
- `ae_use.md`
- `notes`

The engine validates required files and returns `inference_output_invalid` if any are missing/empty.

## Minimal Implementation

```dart
class MyInferenceClient implements InferenceClient {
  @override
  String get id => 'my-provider';

  @override
  bool get isAvailable => true;

  @override
  Future<AeResult<InferenceResponse>> infer(InferenceRequest request) async {
    final payload = <String, dynamic>{
      'ae_install.md': '# install',
      'ae_uninstall.md': '# uninstall',
      'ae_update.md': '# update',
      'ae_use.md': '# use',
      'notes': 'generated by my-provider',
    };

    return AeResult.ok(InferenceResponse(output: payload));
  }
}
```

Plug into core generation:

```dart
final engine = InferenceGenerationEngine(client: MyInferenceClient());
final service = DefaultAeGenerationService(
  templateEngine: const TemplateGenerationEngine(),
  codexEngine: engine,
);
```

Embed in CLI:

```dart
final cli = AeCli(inferenceClient: MyInferenceClient());
```

## Implementation Checklist

1. Respect `InferenceRequest.outputSchema` strictly.
2. Return parseable JSON object output.
3. Surface transport/provider errors through `AeResult.fail(...)`.
4. Implement reliable availability detection (`isAvailable`).
5. Keep output deterministic enough for downstream verify/evaluate flow.

## Notes About Codex Compatibility

Current Codex schema validation is strict.
The generated schema used by AE requires:
- `required` includes every key in `properties`.
- `additionalProperties` is `false`.

If you mirror Codex behavior in another provider, match this strictness to avoid runtime mismatches.
