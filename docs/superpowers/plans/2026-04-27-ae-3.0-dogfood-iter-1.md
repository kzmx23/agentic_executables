# AE 3.0 Dogfood — Iteration 1 Measurement Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve the six open questions left at the end of the Iter 0 dogfood ([§7 of `notes/2026-04-27-ae-3.0-dogfood-iter-0.md`](../notes/2026-04-27-ae-3.0-dogfood-iter-0.md)) by running a structured set of distill experiments on the current `v2` HEAD and emitting a quantitative report.

**Architecture:** This is a **measurement protocol**, not a code change. No files in `agentic_executables_*/lib/` are touched. All experiments run from a freshly-compiled CLI binary on a side branch (`dogfood-iter-1`) that holds artifacts, transcripts, and the final report. The `v2` working tree is restored to its pre-experiment state at the end. Tasks are sequenced so later tasks can reuse outputs from earlier ones (snapshot-and-re-upsert depends on the determinism baseline; the canonical-vs-handwritten-doc comparison depends on the same baseline).

**Tech Stack:** `dart compile exe`, `ae` CLI v3.x, `claude` and `codex` executors, `jq` for envelope parsing, `git` for side-branch artifact storage.

**Output:** A new note at `docs/superpowers/notes/2026-04-27-ae-3.0-dogfood-iter-1.md` with one section per open question and a roll-up that flags any items needing escalation to spec §15 of `[2026-04-17-ae-3.0-design.md](../specs/2026-04-17-ae-3.0-design.md)`.

**Constraints:**
- Do NOT modify any file in `agentic_executables_*/lib/`, `agentic_executables_*/bin/`, or `agentic_executables_*/test/`. If an experiment requires a behavior change, log it as a finding and stop — do not patch.
- Do NOT touch `CHANGELOG.md`. If a finding warrants a release note, the human will land it.
- Do NOT push or open PRs. The side branch stays local. The final commit on `v2` is the report only.
- Pre-existing `.DS_Store` modification on the working tree is unrelated — do not stage it, do not revert it.

## File Structure

**Side branch (`dogfood-iter-1`):**
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q1-determinism/run-{a,b}.json` — distill envelopes from the two same-input runs
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q1-determinism/matrix-{a,b}.yaml` — copies of the resulting `canonical/ae_cli/matrix.yaml`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q1-determinism/diff.txt` — `diff -u` of the two matrices
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q2-scaling/run.json` and `matrix.yaml` for `agentic_executables_core`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q3-upsert/{pre,post}.yaml` and `run.json`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/{env-before.txt,run.json,executor-list.json}`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q5-vs-cli-reference/{matrix.yaml,cli-reference.md.snapshot,disagreements.md}`

**`v2` branch (final commit):**
- Create: `docs/superpowers/notes/2026-04-27-ae-3.0-dogfood-iter-1.md` — synthesis report

Each result file is committed on the side branch with a one-line message (`q<n>: <subject>`) so the final report can link to specific shas.

Expected: `ae canonical snapshot` succeeds with envelope `ok: true`. The directory now has a frozen `v<n>/` subdirectory plus the live one. Record the version number that was bumped.

- [ ] **Step 2: Re-run distill in upsert mode against the same artifact**

Run:
```bash
START=$(date +%s)
/tmp/ae-3.0-iter1 canonical distill --pack agentic_executables_cli --concept ae_cli --mode upsert --root . > /tmp/q3-run.json 2>&1
```

- [ ] **Step 3: Compute the preservation metric**

Run:
```bash
cd docs/superpowers/notes/dogfood-iter-1/iter-1-q3-upsert
awk '/^- id:/ {print $3}' pre.yaml | sort -u > /tmp/q3-pre-ids
awk '/^- id:/ {print $3}' post.yaml | sort -u > /tmp/q3-post-ids
DROPPED=$(comm -23 /tmp/q3-pre-ids /tmp/q3-post-ids | wc -l | tr -d ' ')
ADDED=$(comm -13 /tmp/q3-pre-ids /tmp/q3-post-ids | wc -l | tr -d ' ')
RETAINED=$(comm -12 /tmp/q3-pre-ids /tmp/q3-post-ids | wc -l | tr -d ' ')
cat > findings.md <<EOF
- pre_count: $(wc -l < /tmp/q3-pre-ids)
- post_count: $(wc -l < /tmp/q3-post-ids)
- retained: $RETAINED
- added: $ADDED
- dropped: $DROPPED
EOF
cat findings.md
cd -
```

Expected: `dropped` is the headline number. `dropped == 0` means upsert is preserving (good). `dropped > 0` means features that existed pre-snapshot disappeared after re-distill — a spec violation worth escalating.

- [ ] **Step 2: Run distill against the core pack**

Run:
```bash

timeout 1800 /tmp/ae-3.0-iter1 canonical distill --pack agentic_executables_core --concept ae_core --root . > /tmp/q2-run.json 2>&1
RC=$?
```

Expected: one of three outcomes — (a) success, with a much larger feature_count and proportionally larger wall time; (b) `distillation_failed` with a context-window or token-limit error in the envelope's `error.message`; (c) timeout at 30 min, exit code 124. All three are valid findings — record which.

- [ ] **Step 3: Compute the scaling metric**

Run:
```bash
cd docs/superpowers/notes/dogfood-iter-1/iter-1-q2-scaling
ITER0_CLI_FILES=8
ITER0_CLI_FEATURES=58
ITER0_CLI_WALL=279
CORE_FILES=$(find ../../../../agentic_executables_core/lib -name '*.dart' | wc -l | tr -d ' ')
CORE_FEATURES=$(grep -c '^- id:' matrix.yaml 2>/dev/null || echo 0)
CORE_WALL=$(grep wall_seconds run.json | awk -F: '{print $2}' | tr -d ' ')
cat > findings.md <<EOF
- input_size_ratio (files): $CORE_FILES / $ITER0_CLI_FILES = $(echo "scale=2; $CORE_FILES / $ITER0_CLI_FILES" | bc)
- feature_ratio: $CORE_FEATURES / $ITER0_CLI_FEATURES = $(echo "scale=2; $CORE_FEATURES / $ITER0_CLI_FEATURES" | bc)
- wall_ratio: $CORE_WALL / $ITER0_CLI_WALL = $(echo "scale=2; $CORE_WALL / $ITER0_CLI_WALL" | bc)
- outcome: $(grep '"ok"' run.json | head -1 || echo 'failed/timeout')
EOF
cat findings.md
cd -
```

Expected: a wall_ratio close to or below `input_size_ratio` is sub-linear scaling (the executor compresses well). A wall_ratio much above is a warning sign; a failure is a hard scaling ceiling and goes in spec §15.

## Task 5: Q4 — Dispatcher behavior without `claude` on PATH

**Question:** Without `claude` on PATH (e.g. CI without Claude Code installed), what does the dispatcher do? Iter 0 had both `claude` and `codex` available; Iter 1 must explicitly hide one.

**Files:**
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-before.txt`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-during.txt`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/run.json`
- Create: `docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/findings.md`

- [ ] **Step 1: Capture baseline executor availability**

Run:
```bash
mkdir -p docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude
which claude > docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-before.txt
which codex >> docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-before.txt
echo "PATH: $PATH" >> docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-before.txt
/tmp/ae-3.0-iter1 doctor 2>&1 | tee -a docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-before.txt
```

Expected: `env-before.txt` shows both binaries present plus whatever `ae doctor` reports about executor discovery. (`ae doctor` is documented at [cli-reference.md:235](../../../docs_site/docs/ae-3/cli-reference.md).)

- [ ] **Step 2: Construct a PATH that hides `claude` only**

Run:
```bash
CLAUDE_DIR=$(dirname "$(which claude)")
HIDDEN_PATH=$(echo "$PATH" | tr ':' '\n' | grep -v "^${CLAUDE_DIR}$" | paste -sd: -)
echo "original PATH: $PATH" > docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-during.txt
echo "hidden  PATH: $HIDDEN_PATH" >> docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/env-during.txt
PATH="$HIDDEN_PATH" which claude || echo "claude hidden as expected"
PATH="$HIDDEN_PATH" which codex
```

Expected: `claude` no longer resolves; `codex` still does. If `claude` lives in a directory that also holds other binaries you depend on, this PATH surgery may break something else — abort and pick a different hiding strategy (e.g. temporarily rename the `claude` symlink) and document the choice.

- [ ] **Step 3: Reset and re-run distill under the hidden PATH**

Run:
```bash
rm -rf .ae_hub/canonical/ae_cli
/tmp/ae-3.0-iter1 canonical init --concept ae_cli --title "AE CLI" --root .
START=$(date +%s)
PATH="$HIDDEN_PATH" /tmp/ae-3.0-iter1 canonical distill --pack agentic_executables_cli --concept ae_cli --root . > /tmp/q4-run.json 2>&1
END=$(date +%s)
echo "wall_seconds: $((END - START))" >> /tmp/q4-run.json
cp /tmp/q4-run.json docs/superpowers/notes/dogfood-iter-1/iter-1-q4-no-claude/run.json
```

Expected outcomes (record which):
- (a) Dispatcher selects `codex` automatically. Envelope has `data.executor_used = "codex"`. Distill succeeds.
- (b) Dispatcher fails fast with `distillation_failed` and a clear `no executor available` error.
- (c) Dispatcher tries `claude_code` anyway, child process fails to spawn, error surfaces with a confusing message.

(a) is the desired behavior. (b) is acceptable. (c) is a UX bug for §15.

- [ ] **Step 4: Compose findings**

Run:
